## 1. Функционал MVP (цель и границы)

Источник: **`PLAN.md` §1 «Цель MVP»**, **§2 «Границы MVP»**, **§12 «Definition of Done»**.

### 1.1 Должно работать в итоге

| Область | По ТЗ |
|--------|--------|
| Вход | Авторизация через **Telegram Mini App**; серверная проверка **`initDataRaw`** (§4) |
| Язык | Только **RU** |
| Оболочка | **Fullscreen**, **safe area** / content safe area, единый layout (§2, §4) |
| Темы | Список тем, **поиск по названию**, навигация по структуре темы (§1, этап 4–5) |
| Структура темы | Секции и **чаты**; разделение «описание» vs чат (§3, этап 5) |
| Чаты | **Однотипные экраны чатов** от конфига, не отдельная самописная страница на каждый чат (§3, §5) |
| Сообщения | **Просмотр, создание, редактирование, удаление** (§1, §12) |
| Типы сообщений | **`POST`**, **`TASK`**, **`TASK_ASSIGNMENT`**, **`COMMENT`** с правилами из §3 |
| Медиа | Загрузка **медиа и файлов** (§1, §2) |
| Реакции | Базовые **реакции** (§1, §2) |
| Описание раздела | Просмотр и **редактирование владельцем**; **не чат** (нет комментариев/реакций в ленте) (§3, этап 8) |
| Модуль тем | **Управление темами** (каталог: просмотр / создание / правка / удаление по §9, §12) |
| Архитектура | Внутренняя сессия и пользователь **не только** Telegram id; задел под второй провайдер auth (§1, §10) |

### 1.2 Не входит в MVP (фиксировать, чтобы не раздувать скоуп)

Источник: **`PLAN.md` §2 «Не входит в MVP»**.

- Веб-логин **email/password**
- Полноценная EN-локализация
- Сложная модерация, сложные роли
- Push, оффлайн
- Продвинутые фильтры / аналитика

### 1.3 Критерии «MVP готов» (Definition of Done)

Источник: **`PLAN.md` §12**.

Кратко: стабильный вход из Telegram, поиск/темы, загрузка секций и чатов с backend, полноценный цикл в чатах (CRUD, комментарии где разрешены, TASK / TASK_ASSIGNMENT, реакции, вложения), описание раздела у владельца, CRUD тем в модуле, нет критических блокеров на основных сценариях.

## 3. Экраны по архитектуре плана (FSD)

Источник: **`PLAN.md` §5 «pages»**, этапы **4–9**.

Это **логические страницы приложения**, которые нужно покрыть функционалом (маршруты могут быть сгруппированы иначе):

| Страница (из плана) | Назначение |
|---------------------|------------|
| **`home-page`** | Список тем, поиск, вход в тему / разделы |
| **`theme-page`** | Экран темы: секции и чаты, отдельная обработка описания |
| **`chat-page`** | Универсальная лента + composer (конфиг типа чата) |
| **`section-description-page`** | Просмотр + редактирование описания (может быть 12.1 / 12.1.1 как режимы или маршруты) |
| **`theme-management-page`** | Модуль управления темами (14.x) |

Связка: **theme-page** и **chat-page** в текущей реализации могут частично совпадать с главным («разделы внутри темы»), пока нет отдельного маршрута «тема» — это нужно довести до модели из §5 и этапа 5.
